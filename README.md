# Детекция мошеннических транзакций (Fraud Detection)

Классификация мошеннических банковских операций на сильно несбалансированных данных.

---

## Постановка задачи

Построить и оценить модель бинарной классификации для выявления мошеннических транзакций. Ключевой вызов — сильный дисбаланс классов: мошеннические транзакции составляют менее 1% всех наблюдений.

**Цель:** обеспечить высокое качество обнаружения фрода, делая упор на правильные метрики (PR-AUC) и выбор порога классификации.

---

## Методология

### Ключевые методические решения

- **Главная метрика** — **PR-AUC** (Precision-Recall AUC) из-за сильного дисбаланса классов.
- **Балансировка (SMOTE / веса)** применяется **только к тренировочной выборке** — во избежание утечки данных.
- **Порог решения** выбирается под целевой recall.

### Модели

| Модель | Особенности |
|--------|-------------|
| Logistic Regression | Базовый классификатор для сравнения |
| XGBClassifier | Градиентный бустинг, устойчив к дисбалансу |

### Инструменты

- `pandas`, `numpy` — обработка данных
- `matplotlib`, `seaborn` — визуализация
- `scikit-learn` — модели, метрики, валидация
- `imbalanced-learn` — SMOTE для балансировки
- `xgboost` — XGBoost-классификатор

Лучшая модель по PR-AUC: XGBoost с SMOTE (PR-AUC = 0.8289).

В антифрод-системах выбор порога решающего правила критически влияет на бизнес-показатели. Мы рассмотрели два подхода:

| Подход	| Порог |	Precision	| Recall |	F1 |Пропущено мошенников (FN) |
|---------|-------|-----------|--------|-----|--------------------------|
| Максимизация F1	| 0.9708|	0.937|	0.779|	0.851|	21 из 95 (≈22%)|
|Фиксация recall ≥ 0.85|	0.0042	|0.297|	0.853|	0.440|	14 из 95 (≈15%)|

Данные для порога 0.0042:

+ PR-AUC = 0.8289, ROC-AUC = 0.9553
+ Матрица ошибок: [[56459, 192], [14, 81]]
+ **Потеряно мошеннических транзакций: 14** (при общем количестве мошенников в тесте 95)

## Выводы

- XGBoost с балансировкой SMOTE показал наилучший PR-AUC (0.8289) — эта метрика наиболее релевантна для несбалансированных данных, так как акцентирует внимание на классе меньшинства (фрод). Стандартный ROC-AUC оказался высоким у всех моделей (~0.96), что подтверждает его нечувствительность к дисбалансу и недостаточность для оценки в нашем случае.

- Если приоритет — удержание клиентов (минимизация ложных блокировок), выбираем порог, близкий к максимуму F1.
Если приоритет — минимизация убытков, выбираем порог, обеспечивающий высокий recall, даже ценой большего числа проверок.
Дополнительно можно внедрить калибровку вероятностей и анализ стоимости ошибок (cost-sensitive learning), чтобы порог адаптировался под реальные финансовые потери от FN и операционные издержки от FP.
